## 18. Don’t Build What Already Exists

When I first started building my side project, my instinct was simple.

If I needed something, I built it.

A new function.

A new way to solve the problem.

But building is not always the answer.

Sometimes the solution already exists.

An API.

A library.

Or simply a better way to do it.

So before building something new, I started asking:

**Is there already a tool or approach I can use?**

Building is a skill.

**Knowing when not to build is one too.**
