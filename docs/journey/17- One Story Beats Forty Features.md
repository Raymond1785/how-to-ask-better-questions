## 17. One Story Beats Forty Features

At one point, we had built many features.

Each one worked.

Each one had a reason to exist.

But when everything was shown together, the system became harder to explain.

We could talk about every feature.

But explaining each feature separately did not explain the system as a whole.

What was missing was not another feature.

Once the features became part of the same story, their value became much easier to understand.

The features had not changed.

The story had.

That taught me something important:

**People do not remember a list of features.**

**They remember what the system helped them understand.**
