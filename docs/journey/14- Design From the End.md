## 14. Design From the End

When building something, we often start with:

**“What do we want to build?”**

It sounds like the right question.

But starting there can make us miss things we do not yet know we need.

In information system design, the output often tells us what the input must be.

So instead of starting with the features, start with the result.

**What should the final output tell us?**

Then work backward.

The output reveals what information is needed.

The information reveals what data must be collected.

And the missing data often reveals problems we had not even thought about.

That is why designing from the end matters.

**Starting from what you want to build shows you what you already know.**

**Starting from the end helps you discover what you forgot to ask.**
