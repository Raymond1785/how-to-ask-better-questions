## 8. If They Need to Translate, You Already Lost

We often thought a screen was clear because every field had a label.

The data was correct.  
The logic was correct.  
The result was there.

But people still had to stop and think:

> **“What does this mean?”**

That was the problem.

A user should not have to translate system language into something they can actually use.

We knew what every number meant because we built the system.

But the user did not.

If people need to understand how we think before they can understand what they see, the problem is not the user.

**It is the design.**

The moment people need to mentally translate our system before they can use it, we have already added another task.

Good software should remove that translation.

It should turn data, indicators, and calculations into something people can immediately understand.

We learned that showing information is not enough.

**The real job is to help people understand what the information is telling them.**
